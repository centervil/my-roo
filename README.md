# my-roo: Roocode 業務遂行システム・足場リポジトリ

このリポジトリは、Roocode（旧 Cline）において高度な業務遂行を実現するための「足場（Scaffold）」となる設定ファイル群を提供します。

## コンセプト
- **pmMode 集中管理**: ユーザーは PM とだけ話し、PM が全ての判断と他モードの統制を行います。
- **Mode と Skill の分離**: 役割（Mode）と具体的な手続き（Skill）を分けることで、高い再利用性と拡張性を確保します。
- **自律的品質管理**: PM による「差し戻し（Rejection）」基準を明確化し、プロンプトレベルで品質を制御します。

## 使い方
1. 本リポジトリの内容を、対象プロジェクトの `.roo/` フォルダ内にコピーしてください。
2. Roocode の設定（`custom_modes.json` 等）で、以下のモードを定義してください：
   - `pm`
   - `code`
   - `writer`
   - `reviewer`
   - `qa`
   - `publisher`
3. ユーザーは `pm` モードを選択して対話を開始してください。

## ディレクトリ構造
- `reference.md`: システムの全体像定義
- `policies/`: 共通判断ポリシー
- `skills/`: 標準作業手順書（SOP）
- `rules-{mode}/`: 各モード固有の自動読み込みルール

## 拡張について
このシステムは「足場」です。プロジェクト固有のルールが必要になった場合は、`skills/` に新しい Markdown を追加したり、`rules-{mode}/` 内のファイルを更新したりして、Roocode 自身にシステムを成長させてください。
