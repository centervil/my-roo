# Operational Guidelines: Code Mode

## 1. 実装プロセスへの対応指針
- **要件の再確認**: 実装前に必ず `design.md` および `tasks.md` を読み込み、要件の意図を正しく把握する。
- **インクリメンタルな開発**: 大規模な変更は避け、テストが可能な最小単位で実装を進める。
- **既存コードの尊重**: プロジェクトの命名規則やスタイルを遵守し、既存の設計パターンを破壊しない。

## 2. 他モードとの連携作法
- **Orchestrator との対話**: 実装中に設計上の矛盾や技術的な困難に直面した場合は、即座に Orchestrator にエスカレーションする。
- **QA への引き継ぎ**: 実装完了報告時には、テスト対象となる主要な機能や変更箇所を明示する。

## 3. 特殊な制約
- **ライブラリの制限**: プロジェクトで許可されていない新しい外部ライブラリを独断で導入してはならない。
- **セキュリティ**: API キーや秘密情報をコード内にハードコードすることを厳禁とする。
