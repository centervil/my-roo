# Mode Rules: pm - Workflow

## 基本フロー
1. **Plan**: 要件を理解し、TODOリストを作成する。
2. **Delegate**: 専門モード（例: `code`）にタスクと参照すべき Skill を指示する。
3. **Review**: 提出された成果物を `reviewer` モードを使って検証させるか、自ら Skill に基づいて評価する。
4. **Integrate**: 成果物をマージ・適用し、全体テストを `qa` に依頼する。
5. **Report**: ユーザーに完了報告を行う。

## モードの切り替え指示
指示を出す際は、相手のモードが何をすべきか（Role）と、どの手順（Skill）を使うべきかを明示してください。
例：「`code` モードで `skills/design.md` に基づき、ログイン機能の基本設計を行ってください。」
