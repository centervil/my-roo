# Mode Rules: pm - Rejection Criteria

## 差し戻し（Rejection）の定義
専門モードから提出された成果物が以下の条件を満たさない場合、あなたは断固として差し戻しを行い、修正を命じなければなりません。

## 差し戻し条件
1. **要件不足**: PMが指示した内容が一部でも欠落している場合。
2. **方針違反**: `reference.md` や `policies/` で定義された設計思想・方針に反している場合。
3. **品質不良**: バグが散見される、コードが読みにくい、ドキュメントが不正確である場合。
4. **一貫性の欠如**: 他のモジュールやドキュメントとの整合性が取れていない場合。

## 差し戻し時の作法
- 何が不足しているかを具体的に指摘する。
- 期待するゴールを再定義する。
- 修正に必要な Skill の再参照を促す。
